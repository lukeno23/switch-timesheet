# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Dashboard + Admin.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Scope note:** This phase extends an existing, running dashboard. It does NOT
> introduce a new design system. All tokens, fonts, and component patterns below
> are extracted from the live codebase (`tailwind.config.js`, `src/constants/colors.js`,
> existing shared components). New patterns are introduced only for the four
> surfaces that do not yet exist: Admin CRUD views, Billing entry, Sync status
> header chip, and upcoming-events collapsible sections.

## Spacing Scale

Tailwind's default scale is used at the following step sizes. All values are
multiples of 4 (8-point grid via Tailwind's default `0.25rem = 4px` base):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, badge padding, inline chip padding |
| sm | 8px (`p-2`, `gap-2`) | Compact inputs, form field inner padding |
| md | 16px (`p-4`, `gap-4`) | Default element spacing, card inner padding |
| lg | 24px (`p-6`, `gap-6`) | Section padding, modal body padding |
| xl | 32px (`p-8`) | Modal outer padding, large card internal spacing |
| 2xl | 48px (`py-12`) | Hero/empty-state vertical centering |
| 3xl | 64px (`p-16`) | Full-page-level centering (password gate pattern) |

Exceptions:
- Touch targets (admin table row action buttons): minimum 44px height (`py-2.5` + `text-sm` = ~40px; use `min-h-[44px]` explicitly on icon-only buttons)
- Sidebar nav items: `px-4 py-3` (12px + 16px = established pattern — do not change)
- Scrollbar thumb: 6px (custom CSS, not spacing scale — frozen)

## Typography

All type is set in DM Sans (`font-dm`) except page-level display headings which
may use Playfair Display (`font-playfair`). The existing codebase uses only DM Sans
in practice; Playfair is available but not applied to any running view yet.

| Role | Size | Weight | Line Height | Tailwind classes | Usage |
|------|------|--------|-------------|------------------|-------|
| Body | 14px (`text-sm`) | 400 | 1.5 | `text-sm font-dm` | Table cell content, form help text, modal body copy |
| Label | 12px (`text-xs`) | 700 | 1.4 | `text-xs font-bold uppercase tracking-wider font-dm` | Form field labels, section labels, badge text, sidebar subheadings |
| Heading | 20px (`text-xl`) | 700 | 1.3 | `text-xl font-bold font-dm text-switch-secondary` | Modal titles, card section titles |
| Display | 28px (`text-3xl`) | 700 | 1.2 | `text-3xl font-bold font-dm text-switch-secondary` | View-level page headings (e.g. "Admin", "Billing") |

Notes:
- 14px body is the existing norm in tables and modals (see `TaskDrilldownModal.jsx` `text-sm`)
- 12px labels exactly match the existing form label pattern in `SettingsModal.jsx`
- Do not introduce a 5th size. If a size is not listed here, use the nearest match.
- Numeric values in stat cards use existing `text-4xl font-bold` (`StatCard.jsx`) — frozen, do not change.

## Color

Tokens are defined in `tailwind.config.js` and mirrored in `src/constants/colors.js`.

| Role | Tailwind token | Hex | Usage |
|------|----------------|-----|-------|
| Dominant (60%) | `bg-switch-bg` | `#edf4ed` | Page background, sidebar background (via `bg-white` sidebar on dominant canvas) |
| Secondary (30%) | `bg-white` + `text-switch-secondary` | `#ffffff` surface / `#2f3f28` text | Cards (`rounded-2xl bg-white`), modal surfaces, nav active state (`bg-switch-bg`), all body text |
| Accent (10%) | `switch-primary` | `#a5c869` | Reserved list below |
| Destructive | `red-500` / `red-600` | Tailwind red scale | Deactivate/delete confirmation buttons only |
| Warning (new) | `amber-500` | Tailwind amber scale | Sync-issues chip only — no other use |
| Tertiary / chart use | `switch-tertiary` | `#d2beff` | Chart palette entry — not used for UI chrome |

**Accent (`#a5c869`) is reserved for:**
1. Nav active indicator dot (existing sidebar pattern)
2. Focus ring on form inputs (`focus:border-switch-primary`)
3. Primary CTA buttons that are `bg-switch-secondary` with `hover:bg-switch-secondary-dark` — accent used for hover underline on links only
4. Scrollbar thumb (existing `custom-scrollbar` CSS)
5. Hours value in list cards (`text-switch-primary font-bold`)
6. Form save/confirm buttons (`bg-switch-primary text-white`) in admin modals where the action is non-destructive and non-secondary
7. Loading skeleton pulse accent (if skeleton strategy chosen — see Discretion section)

Accent is **not** used for: general borders, table row hover states, text on white cards, icon fills outside the active-icon pattern.

**Destructive color (`red-500`) is reserved for:**
- "Deactivate" or "Delete billing entry" confirmation button text and icon
- Form validation error messages

**Warning amber is reserved for:**
- The sync-issues chip in the header (amber border + amber text + `AlertTriangle` icon)
- No other use in Phase 3

## New Component Contracts

These patterns do not exist yet. They must be implemented using the rules below.

page heading, not as nested navigation:

### Sync Panel (Admin > Sync tab)

"Sync now" button:
```
bg-switch-secondary text-white text-sm font-bold px-5 py-2.5 rounded-lg
hover:bg-switch-secondary-dark transition-colors flex items-center gap-2 min-h-[44px]
```
Icon: `RefreshCw` (size 16); when syncing, apply `animate-spin`

Sync in-progress state:
- Button disabled, `RefreshCw` spinning
- Progress text below button: `text-sm font-dm text-stone-500` e.g. "Syncing… 7 of 15 switchers done"
- If no per-switcher count available: generic "Sync in progress — this may take a few minutes"

Sync result toast (on completion):
- Floating toast, bottom-right, auto-dismiss after 5s
- Container: `fixed bottom-6 right-6 bg-switch-secondary text-white rounded-xl px-5 py-3 shadow-lg z-50 flex items-center gap-3 animate-in fade-in slide-in-from-bottom-4 duration-300`
- Icon: `Check` (size 18, `text-switch-primary`)
- Text: `text-sm font-dm font-bold`

Fallback after 3 minutes: replace progress text with `text-sm font-dm text-stone-400` "Still running in background — check back shortly"

Sync log table: follows Admin Table Pattern above with columns: Date/Time, Status, Duration, Events Updated, Errors

### Billing Entry Modal

Triggered from Admin > Billing tab "Add entry" or row-click.

Follows Admin CRUD Modal Pattern. Fields:
1. Client selector: `<select>` dropdown (active clients only, sorted A-Z)
2. Month: native `<input type="month">` — chosen over custom dropdown for simplicity (Claude's Discretion)
3. Amount: `<input type="number" step="0.01" min="0">` with currency prefix label
4. Currency: inline radio-style toggle: EUR | USD (two buttons, pill style — active: `bg-switch-secondary text-white`, inactive: `bg-stone-100 text-stone-500`)
5. FX Rate (conditional, shown only when USD selected):
   ```
   label: "FX rate to EUR (e.g. 0.92)"
   input: type="number" step="0.0001" min="0"
   help text: "Rate captured at time of entry — not updated later"
   ```
6. EUR equivalent (computed, read-only when USD): `text-sm text-stone-500 font-dm mt-1` "= €{amount}"
7. Billing type (optional): `<select>` with options: `—` (none), `Retainer`, `Project`
8. Notes (optional): `<textarea rows={2}>` same input styling
9. Entered by: `<input type="text">` free-text label

## Interaction Contracts

### Form Validation Timing

Validate on blur (field leave), not on keystroke. Show errors immediately on blur;
clear errors as soon as the field value changes. Do not validate on initial mount.

### Modal Open/Close

Open: `animate-in fade-in zoom-in-95 duration-200` (existing pattern)
Close: no exit animation — remove from DOM on `isOpen = false` (existing pattern)
Click-outside: close modal (add `onClick` to overlay div with `stopPropagation` on inner container)
Escape key: close modal (add `useEffect` with `keydown` listener)

### Admin Table Row Click

Row click opens the edit modal pre-populated with that entity's data.
Row does NOT navigate to a new view (all admin editing is modal-based, per D-06).

### "Show deactivated" Toggle

When checked: append soft-deleted rows at the bottom of the table, visually separated
by a `text-xs text-stone-400 font-dm py-2 border-t border-stone-100` divider
"Deactivated" label. Inactive-badge styling applies to those rows.

### Sync Now Button State Machine

```
idle → { label: "Sync Now", icon: RefreshCw (static), enabled: true }
  → click → firing
firing → { label: "Starting sync…", icon: RefreshCw (animate-spin), disabled: true }
  → Edge Function responds → polling
polling → { label: "Syncing…", progress text below, disabled: true }
  → sync_runs status = "completed" → done
done → { toast shown, button returns to idle, data refresh triggered }
  → sync_runs status = "failed" → error
error → { toast with red background, text: "Sync failed — check Admin > Sync log", button returns to idle }
  → 3 minutes elapsed without completion → fallback
fallback → { progress text: "Still running in background — check back shortly", button returns to idle }
```

### AI Report Cache Key

Per D-22: cache key = stable hash of (dateRange.start + dateRange.end + sorted active filters + selected entity id).
When key changes, clear report and show "Generate Insights" button again.
When key matches stored key, serve cached report — no new Gemini call.

### Date Range — Default on First Load

On first successful Supabase data load, set `dateRange` to:
- start: `2026-01-04` (anchor date per D-20)
- end: today's date (ISO format)
This gives management the full historical view by default.
