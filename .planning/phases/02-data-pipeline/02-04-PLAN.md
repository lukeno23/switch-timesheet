**Step 2: Create `supabase/functions/sync/index.ts`.**

This is the main sync orchestrator. It serves as a Deno HTTP handler (Supabase Edge Function pattern):

```typescript
import { serve } from "https://deno.land/std@0.168.0/http/server.ts";
```

Wait -- use the current Supabase Edge Function pattern. In Deno 2.x with Supabase, the entry point is:

```typescript
Deno.serve(async (req: Request) => { ... });
```

The function must handle:

**A. Authentication (per D-18):**
- For cron triggers: Check `Authorization: Bearer {service_role_key}` header (pg_cron uses service role key via Vault)
- For manual triggers: Check `x-sync-secret` header against `Deno.env.get("SYNC_SECRET")` (per Research Open Question #2)
- Return 401 if neither matches

**B. Request parsing:**
- Parse JSON body for optional params: `{ trigger: "cron" | "manual", backfill_start?: string, backfill_end?: string }`
- Default trigger is "manual" if not specified

**C. Window calculation (per D-14, D-15):**
- Standard sync: `window_start = today - 7 days`, `window_end = today + 14 days`
- Backfill mode: use `backfill_start` and `backfill_end` from request body (D-15: first sync from Feb 1, 2026)
- Convert to RFC3339 format for Google Calendar API

**D. Reference data loading:**
Load from Supabase at the start of each sync (ensures classification uses latest data per D-04 / ADMN-04):
1. Active switchers: `supabaseAdmin.from('switchers').select('*').eq('active', true)`
2. Active clients: `supabaseAdmin.from('clients').select('*').eq('active', true)`
3. Client aliases: `supabaseAdmin.from('client_aliases').select('alias, clients(name)')`
4. Categories: `supabaseAdmin.from('categories').select('*')`
5. Build alias map using `buildAliasMap()`
6. Build valid client/category/department sets for LLM validation

**E. Create sync_runs record (per D-28):**
Insert a sync_runs row with `started_at = now()`, `status = 'running'`, `trigger`, `window_start`, `window_end`. Store the returned `sync_run_id`.

**F. Per-Switcher pipeline (per D-16):**

Initialize SyncMetrics accumulator. Get service account JSON from `Deno.env.get("GOOGLE_SERVICE_ACCOUNT_JSON")`.

For each active Switcher, wrapped in try/catch for error isolation:

1. **Fetch:** Get Google access token via `getGoogleAccessToken(serviceAccountJson, switcher.email)`. Fetch calendar events via `fetchCalendarEvents(accessToken, switcher.email, windowStartRFC, windowEndRFC)`.

2. **Filter:** Apply `filterEvents()` to remove personal, all-day, zero-duration, weekend events (PIPE-04).

3. **Parse:** For each remaining event, create a `ParsedEvent`:
   - `google_event_id` = event.id
   - `recurring_event_id` = event.recurringEventId ?? null (D-22)
   - `title` = event.summary
   - Parse title using `parseTitle(event.summary)` -> `client_name_raw`, `task_details`
   - `description` = event.description ?? null
   - `attendees` = map event.attendees to simplified objects
   - `start_at` = event.start.dateTime (ISO 8601)
   - `end_at` = event.end.dateTime
   - `duration_minutes` = Math.round((new Date(end_at) - new Date(start_at)) / 60000) (D-23)
   - `event_date` = start_at date portion (YYYY-MM-DD)
   - `day_of_week` = day of week from start_at (0=Mon..6=Sun). Use: `(new Date(start_at).getUTCDay() + 6) % 7` to convert Sunday=0 to Monday=0 system.
   - `off_schedule` = day_of_week === 4 (Friday, per D-11)
   - `temporal_status` = new Date(start_at) > new Date() ? "upcoming" : "completed" (D-24)

4. **Classify (Stage 1 - Rules):** For each parsed event, call `classifyEvent(parsed, switcher.name, switcher.primary_dept, switcher.is_management_member)`. Track metrics: increment rule_classified or misc_count. Track borderline events separately.

5. **Collect unresolved:** Gather events where classification_method === "misc" for LLM fallback.

6. **Hard delete removed events (D-17):** Collect all google_event_ids fetched for this Switcher in this window. Call `supabaseAdmin.rpc('delete_missing_events', { p_switcher_id, p_google_event_ids, p_window_start, p_window_end })`.

7. **Upsert classified events:** For each classified event:
   - Look up `client_id` from clients table by resolved client name (or null if not found / non-client)
   - Look up `category_id` from categories table by category name (or null if Misc)
   - Call `supabaseAdmin.rpc('upsert_event', { ...all fields })` (PIPE-03)

If any step fails for this Switcher, log the error in `metrics.errors[switcher.name]` and continue to the next Switcher (D-16).

**G. LLM Fallback (Stage 2) -- after all Switchers processed (per D-06):**
Collect ALL unresolved (Misc) events across all Switchers. If any exist:
1. Call `classifyWithLLM(unresolvedEvents, geminiApiKey, validCategories, validClients, validDepartments, switcherContext)`
2. Update events in DB with LLM classification results (classification_method = "llm")
3. Update metrics: increment llm_classified for resolved, keep misc_count for still-unresolved

**H. LLM Audit Pass (Stage 3) -- per D-07, D-08:**
Collect events for audit: ALL borderline rule-classified events + 10% random sample of confident rule-classified events.
1. Call `auditRuleClassifications(eventsToAudit, geminiApiKey, validCategories, validClients, validDepartments)`
2. For disagreements: update event classification_method to "llm_corrected", update category/department
3. Write audit_log entries for corrections
4. Update metrics: increment llm_corrected

**I. Update sync_runs record (per D-28, D-29, PIPE-06):**
Update the sync_runs row with: `completed_at = now()`, `status = 'success' | 'partial' (if any errors) | 'failed'`, all metric counts, errors JSONB, `duration_ms = now - started_at`.

**J. Return classification summary report (per D-26):**
Return JSON response with:
```json
{
  "status": "success" | "partial",
  "sync_run_id": "...",
  "window": { "start": "...", "end": "..." },
  "metrics": { "events_processed": N, "rule_classified": N, "llm_classified": N, "llm_corrected": N, "misc_count": N, "borderline_count": N },
  "errors": { "SwitcherName": "error message" },
  "duration_ms": N
}
```
