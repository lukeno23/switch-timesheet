**Step 1: Create `supabase/functions/_shared/eventFilter.ts`** (per D-12, D-11, PIPE-04)

Port the personal event filter from `classify_with_ai.py` (the more complete version per Research Pitfall 7). The filter must:

1. **isPersonalEvent(title: string): boolean** -- Check against two lists:
   - `PERSONAL_EXACT`: Exact title matches (case-insensitive). Port the complete list from `classify_with_ai.py` including: "lunch", "gym", "doctor", "dentist", "break", "personal", "physio", "pilates", "yoga", "nap", "off", "leave", "sick", "haircut", "errands", "hospital", "surgery", "gynae", "ultrasound", "pick up pips", "stay with kids", "slow vinyasa", "had to lie down", "hygiene walk", "luncch", "malta marathon", "p&p flea" -- read the actual file for the COMPLETE list.
   - `PERSONAL_CONTAINS`: Substring matches (case-insensitive). Port from `classify_with_ai.py`: "personal", "doctor", "dentist", "sick", "leave", "break", "lunch", "gym", "pilates", "yoga", "haircut", "errands" -- read the actual file for the COMPLETE list.

2. **isAllDayEvent(event: GoogleCalendarEvent): boolean** -- Returns true if event has `start.date` instead of `start.dateTime` (Google Calendar API convention for all-day events).

3. **isZeroDuration(startAt: string, endAt: string): boolean** -- Returns true if start equals end.

4. **isWeekendEvent(dayOfWeek: number): boolean** -- Returns true for Saturday (5) or Sunday (6) using 0=Mon convention.

5. **filterEvents(events: GoogleCalendarEvent[]): GoogleCalendarEvent[]** -- Applies all filters: removes personal, all-day, zero-duration, and weekend events. Returns filtered array.

Note: Friday events (dayOfWeek=4) are NOT filtered -- they pass through with `off_schedule: true` set later during parsing.

**Step 2: Create `supabase/functions/_shared/titleParser.ts`** (per D-21 -- Claude's discretion: parse at classification-time)

Parse pipe-delimited titles per the Switch convention "Client | Task Details":

```typescript
export interface ParsedTitle {
  clientRaw: string;
  taskDetails: string;
}

export function parseTitle(summary: string): ParsedTitle
```

Rules from `instructions.md`:
- Split on first pipe `|` character
- Left side = client name (trimmed), right side = task details (trimmed)
- If no pipe, entire title is task_details, clientRaw is empty string
- If multiple pipes, first split only (rest stays in task_details)
- Handle edge case: title is just a pipe character -> both empty

**Step 3: Create `supabase/functions/_shared/aliasResolver.ts`** (per CLAS-04)

Two functions:

1. **resolveClientAlias(clientRaw: string, aliasMap: Map<string, string>): string** -- Looks up clientRaw (case-insensitive) in the alias map. Returns canonical name if found, otherwise returns clientRaw. The aliasMap is loaded from DB at sync time (Plan 01 seeded the data).

2. **isNonClientName(name: string): boolean** -- Returns true for non-client names that appear in the client field: "Marketing", "PM", "Design", "HR", "Admin", "BD", "TBC", "SAFETY TIME", "Switch", "Internal". These should not be treated as client names during classification. Read `instructions.md` "Non-Client Names That Appear in the Client Field" section for the COMPLETE list.

3. **buildAliasMap(aliases: Array<{ alias: string; client_name: string }>): Map<string, string>** -- Builds a case-insensitive lookup Map from DB results. Keys are lowercase alias strings, values are canonical client names.
