## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`, `p-1`) |
| sm | 8px | Compact element spacing (`px-2`, `py-2`) |
| md | 16px | Default element spacing (`p-4`, `gap-4`) |
| lg | 24px | Section padding (`p-6`) |
| xl | 32px | Layout gaps (`p-8`) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets for icon-only buttons: 44px minimum (accessibility requirement)
- Inline `py-1.5` (6px) used in ToggleGroup and DropdownFilter — preserve this value; it is already in use throughout the codebase

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#edf4ed` | Page background, icon container fills, muted surfaces |
| Secondary (30%) | `#2f3f28` | Sidebar/nav, headings, primary button background, chart color 1 |
| Accent (10%) | `#a5c869` | See reserved list below |
| Tertiary | `#d2beff` | Lavender — chart color 4, accent report label overlays |
| Destructive | `#ef4444` (Tailwind `red-500`) | Error state text only; no destructive actions in Phase 1 |

Accent (`#a5c869`) reserved for:
- Active/focused border on form inputs
- Primary CTA hover state (background fill on hover)
- Stat sub-value text (e.g. hours/day)
- Inline link color (settings modal, AI report)
- Spinner/loader icon (`Loader2`)
- Checkmark icon in multi-select dropdowns
- Sort indicator arrows
- Border accent on the CSV upload card (`border-t-4`)

**NOT used for:** general body text, nav items, or card backgrounds.

**Phase 1 addition — Password Gate color:**
- Gate background: `#edf4ed` (dominant — matches app background)
- Gate card: `#ffffff` with `border-t-4 border-[#a5c869]` (matches CSV upload card pattern)
- Input focus border: `#a5c869` (matches settings modal input pattern)
- Submit button: `bg-[#2f3f28]` text-white (matches primary button pattern)

Source: `src/App.jsx` COLORS constant + all className usage patterns.

**Phase 1 note:** All hardcoded hex values must be extracted to `tailwind.config.js` under `theme.extend.colors` (requirement QUAL-03). Token naming convention: semantic names prefixed with `switch-` (e.g. `switch-bg`, `switch-primary`, `switch-secondary`, `switch-tertiary`). This balances readability with brand clarity.

Proposed token map:
| Token name | Hex | Old className |
|---|---|---|
| `switch-bg` | `#edf4ed` | `bg-[#edf4ed]` |
| `switch-primary` | `#a5c869` | `text-[#a5c869]`, `bg-[#a5c869]`, `border-[#a5c869]` |
| `switch-secondary` | `#2f3f28` | `text-[#2f3f28]`, `bg-[#2f3f28]` |
| `switch-tertiary` | `#d2beff` | `text-[#d2beff]` |
| `switch-secondary-dark` | `#1a2416` | `bg-[#1a2416]` (hover state) |

## Interaction Contract

### Password Gate (DASH-03)

- Rendered as a full-screen centered layout matching the existing CSV upload screen pattern (logo + card)
- Card: `bg-white rounded-2xl shadow-sm border border-stone-100 border-t-4 border-[#a5c869] p-8 max-w-md w-full`
- Switch logo (`LogoMain`) displayed above the card with `mb-12`
- Single password `<input type="password">` with `font-dm` and `focus:border-switch-primary` styling
- Submit button: full-width, `bg-switch-secondary text-white`, consistent with "Select CSV File" button
- On submit: SHA-256 hash the input client-side, compare to `VITE_APP_PASSWORD_HASH` env var
- Authenticated state persists in `sessionStorage` — not `localStorage` (session-scoped per decision D-04)
- On match: render the app. On mismatch: show inline error below the input, clear the input field, keep focus on input
- No "forgot password" link, no username field, no remember-me checkbox (out of scope)
- The gate wraps at `src/main.jsx` level or as the first guard inside `<App />` — either is valid; preserve StrictMode wrapper

### Error Boundaries (QUAL-04)

- Each top-level route/view wrapped in an `<ErrorBoundary>` component
- Fallback UI: centered card with heading + body + "Refresh page" button
- Same visual language as the upload screen (centered, card, logo optional)
- `console.error` the caught error for debugging; no external error reporting service

### Tailwind Token Migration (QUAL-03)

- All `bg-[#hex]`, `text-[#hex]`, `border-[#hex]` arbitrary values replaced with `bg-switch-*` / `text-switch-*` / `border-switch-*` tokens
- `font-dm` and `font-playfair` replaced with `font-sans` and `font-serif` mapped to the Google Fonts families via `tailwind.config.js`
- Existing Tailwind utility classes (`rounded-`, `shadow-`, `p-`, `gap-`, etc.) are NOT changed
