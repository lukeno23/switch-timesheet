## Spacing Scale

Declared values (multiples of 4 only). Match existing Tailwind utility usage patterns
observed in `TaskTable.jsx`, `TaskDrilldownModal.jsx`, `Card.jsx`, and `WeeklyCalendar.jsx`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline badge padding, sort-icon margin |
| sm | 8px | Compact cell padding, chip inner padding, filter dropdown item spacing |
| md | 16px | Default card inner padding, table row `px-4`, section header gaps |
| lg | 24px | Card `p-6` outer padding, modal header/body padding, list view `mb-6` |
| xl | 32px | Section-level top/bottom margin, calendar grid gap container |
| 2xl | 48px | Major vertical section breaks between distinct content blocks |
| 3xl | 64px | Page-level top padding on mobile |

Exceptions:
- Calendar event block height: proportional to event duration. 1 hour = 48px min-height, scaling linearly. Minimum block height 20px for sub-15-minute events.
- Touch targets on prev/next week navigation buttons: minimum 44px height.
- Filter dropdown checkbox rows: minimum 36px height for pointer accuracy.
- Override dropdown select controls: minimum 36px height.

Source: existing component patterns. No deviations from 8-point base scale for non-exception elements.

## Interaction Contracts

### Chart Click â†’ Modal (D-01, D-02, D-03)

- Every `<Bar>`, `<Cell>` in `<Pie>`, `<Area>` segment, and `<Line>` data point gets an `onClick` handler.
- Handler signature: `(entityName, filteredEvents) => openChartModal(entityName, filteredEvents)`
- Modal: `ChartDrilldownModal` with entity name displayed, total hours, event table.
- Modal header entity name is a clickable button. `onClick` fires `onNavigate` to the entity's detail page and closes the modal.
- Cursor: `cursor: pointer` on all clickable chart elements via Recharts `style` prop.

### Historical / Upcoming Toggle (D-12, D-13, D-14, D-15)

- Toggle sits in the top navigation bar, right of the date range controls.
- Two states: "Historical" (default) | "Upcoming"
- State stored in `App.jsx` as `const [upcomingMode, setUpcomingMode] = useState(false)`.
- All views (Dashboard, Switchers, Clients, Teams, Categories) receive `upcomingMode` prop.
- In upcoming mode: data filtered to `event.dateObj >= today`. All charts, stats, and breakdowns recalculate from this filtered dataset. No flat-list fallback.
- Toggle renders as a pair of pill buttons: inactive pill = `bg-white text-stone-400 border border-stone-200`; active pill = `bg-switch-primary text-white`.
- `UpcomingEvents.jsx` component is removed from all views when this toggle ships.

### Weekly Calendar (D-04 through D-08)

- Columns: Mon, Tue, Wed, Thu only (Switch 4-day work week).
- Navigation: left arrow button (prev week) + right arrow button (next week) + centered week label ("Week of DD MMM YYYY"). Arrow buttons are `ChevronLeft` / `ChevronRight` lucide icons, `size={16}`, minimum 44px touch target.
- Event blocks: background color from `COLORS.chartPalette` keyed by client name index. Text: client name in `font-bold text-xs` + category name below in `text-xs text-stone-600 truncate`.
- Block height: `Math.max(20, (minutes / 60) * 48)` pixels as inline style.
- Day column >8h warning: day header `div` background changes to `bg-amber-50 border border-amber-200` for 8â€“10h, `bg-red-50 border border-red-200` for >10h.
- Event block click: fires `onEventClick(event)` â†’ opens `ChartDrilldownModal` pre-populated with that single event. Modal close returns to calendar.
- Calendar lives as a collapsible section within `DetailView` when `type === 'switcher'`. Section toggle via chevron, default open.

### Table Column Filtering (D-09, D-10, D-11)

- Column headers retain existing sort-click behavior.
- Each categorical column (Team, Switcher, Client, Category) header gets a `Filter` lucide icon (`size={12}`) inline after the sort icon.
- Clicking the filter icon opens `FilterDropdown` as an absolutely-positioned popover below the header cell.
- FilterDropdown: checkbox list of all unique values in that column. Checked = included. "Clear" button at bottom resets to all-checked.
- Date column: filter icon opens a date-range mini-picker (two `<input type="date">` fields, min/max).
- Active filter state: filter icon changes from `text-stone-300` to `text-switch-primary` when a filter is active on that column.
- Filtered row count displayed as `text-xs text-stone-400` below the table: "Showing {n} of {total} events".

### Classification Override (D-22, D-23, D-24)

- `ClassificationOverridePanel` renders inside the floating task modal (ChartDrilldownModal / TaskDrilldownModal) as an expandable section at the bottom of each event row.
- Trigger: small "Edit" icon button (`Pencil` lucide, `size={14}`) at the far right of each event row in the modal. Only visible on row hover.
- Expanded state shows three `<select>` dropdowns: Client, Category, Department. Pre-populated with current classification values.
- "Save" button: `bg-switch-primary text-white text-xs px-3 py-1 rounded-lg`. Fires API call to admin Edge Function.
- "Cancel" text button: `text-stone-400 text-xs`. Collapses panel without saving.
- After successful save: row shows a `user_override` badge (`text-xs bg-purple-50 text-purple-700 border border-purple-200 rounded px-1`). Toast notification confirms save.
- Override badge: same badge style as `DEPT_COLORS` pattern, purple variant, reads "Overridden".
